
# 🗄️ AdventurerTavern 数据库管理指南

**AdventurerTavern** 采用 **IndexedDB** 技术将所有游戏数据（用户账号、存档、游戏状态）存储在客户端（即访问者的浏览器）中。

由于数据不经过服务器存储，作为开发者或管理员，您无法远程查看玩家的数据。但您可以使用浏览器自带的**开发者工具**来查看、修改或重置您当前浏览器中的游戏数据。

本指南将介绍如何操作 `AdventurerTavernDB` 数据库。

---

## 🛠️ 如何进入数据库管理界面

以下步骤以 **Google Chrome** 或 **Microsoft Edge** 为例（Firefox 操作类似）：

1.  **打开游戏**：在浏览器中运行游戏。
2.  **打开开发者工具**：
    *   按键盘上的 `F12` 键。
    *   或者：右键点击页面任意位置，选择 **“检查” (Inspect)**。
3.  **定位存储面板**：
    *   在开发者工具顶部的标签栏中找到 **Application** (应用) 标签。
    *   *注意：如果屏幕较窄，可能需要点击 `>>` 图标才能看到 Application。*
4.  **展开 IndexedDB**：
    *   在左侧侧边栏中，找到 **Storage** (存储) -> **IndexedDB**。
    *   点击展开箭头，您应该能看到 **`AdventurerTavernDB`**。

---

## 📂 数据库表结构说明

点击 `AdventurerTavernDB` 下的表名，右侧将显示该表的所有数据记录。

| 表名 (Table) | 说明 | 关键字段 |
| :--- | :--- | :--- |
| **`users`** | 存储用户注册信息 | `id` (UID), `username`, `password` (原型阶段为明文) |
| **`saves`** | 核心存档记录（金币、时间、位置等） | `id` (存档ID), `userId` (所属用户), `slotId` (0=自动, 1-3=手动) |
| **`savedInventory`** | 存档关联的道具库存 | `saveId` (关联存档), `itemId`, `count` |
| **`savedCharacters`** | 存档关联的角色状态（好感/等级） | `saveId`, `charId`, `affinity`, `level` |
| **`savedFacilities`** | 存档关联的设施建设等级 | `saveId`, `facilityId`, `level` |
| **`savedRevenueLogs`** | 存档关联的营收/流水记录 | `saveId`, `amount`, `type` |

## ⚡ 常用操作指南

### 1. 查看最新数据
IndexedDB 的视图不会自动实时更新。
*   点击表名上方的 **刷新图标 (Refresh)** 🔄，以加载游戏运行中产生的最新数据。

### 2. 修改游戏数值 (作弊/调试)
您可以直接修改数值来测试游戏功能（例如增加金币）。
1.  点击 `saves` 表。
2.  找到您的存档记录（可以通过 `userId` 和 `slotId` 辨别）。
3.  点击 `value` 列中的对象展开它。
4.  双击 `gold` 的数值，输入新的金额（例如 `999999`），然后按回车保存。
5.  **重要**：修改后需要**刷新游戏页面** (F5) 才能生效，因为游戏会在加载时读取数据库。

### 3. 删除特定数据 (如删除用户)
1.  点击 `users` 表。
2.  选中要删除的用户行。
3.  按键盘上的 `Delete` 键，或点击工具栏上方的 `X` 图标。
4.  *注意：仅删除用户不会自动删除该用户的存档数据（除非在代码层面做了级联处理），手动管理时需注意数据残留。*

### 4. 重置游戏 (删库跑路)
如果您遇到严重 Bug 或想彻底重置游戏回到初始状态：
1.  在左侧侧边栏点击 **`AdventurerTavernDB`** (数据库名称)。
2.  在右侧面板中，点击 **"Delete database" (删除数据库)** 按钮。
3.  刷新页面，游戏将重新初始化，所有用户和存档将被清空。

### 5. 修改用户密码 (管理员/找回密码)
由于本项目目前处于原型阶段，为了方便调试，密码是以**明文**形式存储的，您可以直接修改：
1.  点击 `users` 表。
2.  找到需要修改密码的用户行。
3.  双击 `password` 列的值。
4.  输入新的密码并按回车保存。
5.  下次登录时请使用新密码。

---

## ⚠️ 注意事项

*   **数据本地性**：此操作仅影响**当前浏览器**。您无法通过此方法管理其他玩家的数据。
*   **隐私模式**：如果您在浏览器的“无痕/隐私模式”下运行游戏，关闭窗口后 IndexedDB 数据会被浏览器自动清除。
*   **数据安全**：手动修改数据库结构（如删除关联键）可能导致游戏加载报错，请谨慎操作。
